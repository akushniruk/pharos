# Governance patterns

This page tracks **documentation intent** for policies, allowlists, and “show your work” defaults. Copy must stay honest: label **roadmap** where product behavior is not shipped yet.

## Pillars (documentation)

- **Observable by default** — Prefer traces humans can inspect without vendor-only tooling.
- **Explicit policy** — When enforcement exists, surface it in UX and reference docs together.
- **Proof for shipping** — Tie releases to verifiable steps (see [Releases and upgrades](/docs/guides/releases-and-upgrades)).
