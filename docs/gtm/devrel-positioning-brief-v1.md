# DevRel positioning brief (v1) — Pharos

**Strategic lock (CEO):** **Security-led primary**, **velocity secondary.** All hero messaging and pillar order follow **trust, control, observability** first; **speed** only as *confident velocity* after visibility.

## Positioning vs. generic “agent tools”

| Generic agent tools | Pharos |
| ------------------- | ------ |
| Optimize for raw output and feature breadth | Optimize for **observability**, **governability**, and **understandability** of agent behavior |
| “Move fast, apologize later” as subtext | **Prove what ran** — speed follows when teams can inspect and approve |
| Black-box or chat-centric UX | Execution narrative: **what ran, in what order, under what rules** |
| “Trust the model” | **Trust the system you can inspect** — humans stay in the loop by design |
| One-size onboarding | **Onboarding as curriculum** for observable, policy-aware agent habits |

**Elevator (external):** Pharos helps teams adopt AI coding agents **without losing visibility, auditability, or control** — so security and platform leaders can endorse rollout and developers still move fast **with proof**.

## Three content pillars

1. **Trust, governance, and auditability for practitioners** — Lead here: practical patterns for visibility, allowlists, explicit prompts, “show your work” defaults, and reproducible runs. Frame productivity as the **downstream** win once the bar is met.
2. **Lights-on engineering** — Tutorials that show *observable* agent workflows: tracing a change from intent → tool use → artifact, with emphasis on reproducibility and clarity.
3. **Build in public / shape the standard** — Early access stories, RFC-style posts on agent UX expectations, and community input on what “observable agent” should mean in real IDEs and repos.

## Minimal quarterly theme (Q2-style placeholder)

**Theme:** *“Trust, then throttle — onboarding to observable, governable agents.”*

- **Month 1:** Narrative + **pillar 1** content live in-repo (README + at least one `docs/` anchor); same story echoed on X with links to those paths.
- **Month 2:** 2–3 technical posts demonstrating **policy-aware, reproducible** workflows end-to-end.
- **Month 3:** Community recap — what we learned from security/platform questions; tighten messaging from real objections.
