# Pharos — public positioning

Pharos is positioned as **security-led first**, **velocity second**. The product makes AI coding agents **observable and understandable** on the machines where software is built.

**Narrative hierarchy:** Lead with **trust, control, and safety** (what is running, how you govern it, auditability). Treat **speed** as *confident velocity once visibility is in place* — not the headline before clarity.

## Who it is for

1. **Security-, compliance-, and platform-governed engineering organizations** — people who must answer what ran, under what policy, and whether it can be proven before widening agent rollout.
2. **Technical leads and staff engineers** who balance developer experience with security review and need agents to meet existing accountability bars.
3. **Early-adopter builders and OSS maintainers** who care about transparency and control.

## The problem

AI coding agents are becoming default in daily development. Teams want speed but often get **opaque automation**: unclear prompts, hidden tool use, hard-to-audit actions, and uncertainty about what is actually running locally. **Trust and approval** stall first; velocity cannot scale when leadership cannot **see**, **explain**, and **govern** agent behavior the same way they do for CI, dependencies, and infrastructure.

## Differentiation

Pharos is not another chat client for code. It is the layer that makes **agent execution observable and governable** where real software is built — so security and platform leaders can say **yes** to agents **without giving up understanding or accountability**. Productivity shows up as **confident** speed, not as a trade against visibility.

**One-line promise (external):** *See what your coding agent does, govern it, and prove it — then ship faster because the system is inspectable.*

**Supporting line (velocity, secondary):** *Confident speed comes after the lights are on.*

## Proof themes

1. **Auditability first** — Agent actions can be surfaced so practitioners and reviewers can scan, share, and reproduce them.
2. **Observable by default** — What ran, in what order, under what rules is a first-class story.
3. **Local-first clarity** — Execution is understandable in *your* environment.
4. **Control as enabler of scale** — Guardrails are framed as what lets more people use agents safely.
